python+flask测试web服务

最近领导交给我一个任务：通过flask框架测试web服务，那咋测啊？？
领导给我的思路是先编写一个flask的包含业务逻辑python接口文件，然后把这个flask的文件运行起来之后将python文件中需要的参数通过postman发送到服务器上，这样就可以通过postman上发送的请求调用对应的接口处理不同的业务了。
所以在这个过程中flask我的理解是充当一个任务分配的角色，将客户端那边的请求分发给各自的接口来调用对应的处理函数。而postman则是模拟浏览器或者客户端给服务器发送请求的。

整个过程是这样的：
1.用户请求（发送过来一个URL）；
2.flask解析用户请求（URL）；
3.找到这个url所代表的函数；
4.执行此函数，得到结果，返回给客户端；
5.客户端渲染结果，展示给用户

那么这篇博客来说说怎么编写这个flask的py文件：
Flask是由python开发的web框架，框架的作用就是根据浏览器或者客户端发送的请求去分配任务， 不同的任务由不同的函数或者端口处理，这些函数中包含了业务逻辑。在知道了flask框架是干啥的之后，编写flask的py文件也就不是问题了。也就是说这个py文件中必须包含一些函数，这些函数可以处理不同的业务逻辑。那么咱们还是从最简单的hello world开始吧～

所以总结一下，一个flask的python文件包含了以下几个部分：
1.导入flask.Flask类；（From flask import Flask）
2.实例化Flask对象（app=Flask(name)）；
3.绑定路由；（@app.route()整个函数）
4.启动路由/服务器（app.run()）

———–补充———–

flask可以通过python直接启动，也可以通过终端启动：
可以参考
https://blog.csdn.net/LeechengLove/article/details/106724137
步骤如下：
1.启动终端；
2.进入到你的关于flask的py文件目录下，比如我的hello_world.py在D:/flask/test/hello_world.py下，那么终端cd D:/flask/test确保进入你的py文件所在目录下即可；
3.进入之后，直接输入
set FLASK_APP=hello_world.py#指定flask启动的py文件
python-m flask run–host=127.0.0.1#启动flask并制定ip
这个host你也可以自己指定，只要能启动就OK
